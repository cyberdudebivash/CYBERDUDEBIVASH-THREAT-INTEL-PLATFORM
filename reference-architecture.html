COMPONENT DETAIL

Key Component Specifications

🔄 Pipeline Orchestrator

45-stage CI/CD pipeline running on GitHub Actions. Feed ingestion every 6–8 hours with automated retry on failure. Pipeline health monitored with Slack/Telegram alerting.

  • GitHub Actions runners (hosted)
  • Idempotent feed re-ingestion
  • Regression immunity suite (24 test cases)
  • Auto-rollback on quality gate failure

🤖 AI Enrichment Engine

Processes each advisory through classification, MITRE ATT&CK mapping, actor attribution, and detection rule generation. Quality scores assigned before publish.

  • CVSS v3.1 + EPSS enrichment
  • ATT&CK v15 tactic/technique ID validation
  • Sigma + KQL + Suricata rule templates
  • Confidence score (0.0–1.0)

🔍 IOC Quality Governor v1.1

Three-step classifier: (1) unambiguous type match, (2) artifact rejection, (3) network/email IOC parsing. 22 valid IOC types; false-positive rate target <5%.

  • IPv4/IPv6/CIDR, Domain, URL
  • MD5/SHA1/SHA256/SHA512
  • CVE ID, MITRE TTP, Registry Key
  • Filename, ASN, GUID, Mutex

📡 STIX 2.1 / TAXII 2.1 Layer

All enriched advisories serialised as STIX 2.1 bundles containing Indicator, Malware, AttackPattern, ThreatActor, and Vulnerability objects with full relationship graphs.

  • TAXII 2.1 collection endpoint
  • HTTPS transport binding
  • Supports MISP, OpenCTI, Anomali
  • SHA-256 per-bundle content hash

🔐 Security Control Layer

Defence-in-depth across all platform layers: TLS 1.3 in transit, AES-256 at rest, RBAC + MFA, Cloudflare DDoS, CSP Level 3, HSTS, audit logging.

  • bcrypt cost-12 for API key storage
  • HMAC-SHA256 request signing
  • IP CIDR allowlisting (Enterprise)
  • 90-day audit log retention

🏗️ Infrastructure

AWS ap-south-1 (Mumbai) for compute and primary storage. Cloudflare for CDN, WAF, and DDoS mitigation. Daily config backups. DR runbook with automated recovery.

  • AWS ap-south-1 (India region)
  • Cloudflare CDN + WAF
  • S3 for STIX bundle storage
  • 99.9% monthly uptime SLA (Enterprise)
