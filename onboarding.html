API Key Provisioning
⏱ ~5 min
Request Your Enterprise API Key Required

Enterprise API keys are provisioned by the SENTINEL APEX team within 1 business day of subscription confirmation.

ℹ️
Enterprise keys include: 50,000+ API calls/month, STIX 2.1 TAXII access, white-label endpoints, and dedicated rate limit pools separate from free-tier users.
↗ Request API Key
Store Your API Key Securely

Store your API key in an environment variable — never hardcode it in source files.

# Linux / macOS — add to ~/.bashrc or ~/.zshrc export SENTINEL_API_KEY="sk_enterprise_YOUR_KEY_HERE" # Windows PowerShell $env:SENTINEL_API_KEY = "sk_enterprise_YOUR_KEY_HERE" # Docker / Kubernetes env: - name: SENTINEL_API_KEY valueFrom: secretKeyRef: name: sentinel-secrets key: api-key

First API Call
⏱ ~10 min
Verify Your Connection

Run this to verify your API key and check your subscription tier:

# Health + auth check curl -H "X-API-Key: $SENTINEL_API_KEY" \ https://intel.cyberdudebivash.com/api/v1/health # Expected response: { "status": "operational", "tier": "enterprise", "calls_remaining": 49987, "version": "v148.0" }
Fetch Recent CRITICAL Advisories
# Fetch 10 most recent CRITICAL advisories curl -H "X-API-Key: $SENTINEL_API_KEY" \ "https://intel.cyberdudebivash.com/api/v1/advisories?severity=CRITICAL&limit=10" # Python SDK example import requests client = requests.Session() client.headers["X-API-Key"] = os.environ["SENTINEL_API_KEY"] resp = client.get( "https://intel.cyberdudebivash.com/api/v1/advisories", params={"severity": "CRITICAL", "limit": 10, "stix": "true"} ) advisories = resp.json()["advisories"]

# Step 1: Add TAXII connector in Sentinel # Navigate: Sentinel → Data Connectors → TAXII (Preview) Friendly Name: SENTINEL APEX Threat Intel API Root URL: https://intel.cyberdudebivash.com/api/v1/taxii/ Collection ID: enterprise-stix-bundle Username: enterprise Password: $SENTINEL_API_KEY Lookback period: 7 days # Step 2: Import KQL detection rules via API $rules = Invoke-RestMethod ` -Uri "https://intel.cyberdudebivash.com/api/v1/rules/kql/latest" ` -Headers @{"X-API-Key" = $env:SENTINEL_API_KEY} # Then import via Sentinel → Analytics → Import

STIX 2.1 / TAXII 2.1 Feed Setup
⏱ ~30 min
TAXII 2.1 Server Endpoints
# Discovery endpoint GET https://intel.cyberdudebivash.com/api/v1/taxii/ # API Root GET https://intel.cyberdudebivash.com/api/v1/taxii/enterprise/ # Collections GET https://intel.cyberdudebivash.com/api/v1/taxii/enterprise/collections/ # Get STIX objects from collection GET https://intel.cyberdudebivash.com/api/v1/taxii/enterprise/collections/stix-bundle/objects/ ?added_after=2026-05-01T00:00:00Z &match[type]=vulnerability,indicator,threat-actor # Authentication: HTTP Basic Authorization: Basic base64(enterprise:$SENTINEL_API_KEY)
Python TAXII Client Example
from taxii2client.v21 import Server, ApiRoot import os server = Server( "https://intel.cyberdudebivash.com/api/v1/taxii/", user="enterprise", password=os.environ["SENTINEL_API_KEY"] ) api_root: ApiRoot = server.api_roots[0] collection = api_root.collections[0] # Fetch all STIX 2.1 objects bundle = collection.get_objects() for obj in bundle.get("objects", []): print(f"{obj['type']}: {obj.get('name', obj['id'])}")

6
White-Label Configuration
Enterprise MSSP
White-Label Setup Checklist
  • Submit brand assets: logo (PNG/SVG, min 200px), primary color hex, organization name
  • Configure custom domain (optional): point CNAME to intel.cyberdudebivash.com
  • Request white-label PDF template activation from your account manager
  • Test white-labeled advisory export with a sample CVE
  • Configure multi-tenant API keys for each client (via admin API)
  • Set up per-client rate limit pools and access controls
White-Label API Usage
# Fetch white-labeled PDF dossier (your branding applied) curl -H "X-API-Key: $SENTINEL_API_KEY" \ -H "X-White-Label-Org: your-org-slug" \ "https://intel.cyberdudebivash.com/api/v1/advisories/CVE-2026-XXXX/pdf" \ -o branded_dossier.pdf # Create sub-tenant API key (MSSP admin endpoint) curl -X POST \ -H "X-API-Key: $SENTINEL_API_KEY" \ -H "Content-Type: application/json" \ -d '{"client_name":"Client-A","tier":"enterprise","rate_limit":5000}' \ "https://intel.cyberdudebivash.com/api/v1/admin/tenants"

Monitoring & Alerting Setup
⏱ ~30 min
Webhook Configuration

Configure webhooks to receive real-time notifications when new CRITICAL or HIGH advisories are published:

# Register webhook endpoint curl -X POST \ -H "X-API-Key: $SENTINEL_API_KEY" \ -H "Content-Type: application/json" \ -d '{ "url": "https://your-soc.com/sentinel-apex-webhook", "events": ["advisory.critical", "advisory.high", "kev.new"], "secret": "your-webhook-secret", "severity_filter": ["CRITICAL", "HIGH"] }' \ "https://intel.cyberdudebivash.com/api/v1/webhooks" # Webhook payload structure (advisory.critical event) { "event": "advisory.critical", "advisory_id": "CVE-2026-XXXXX", "cvss": 9.8, "epss": 0.94, "kev": true, "published_at": "2026-05-11T22:30:00Z", "stix_bundle_url": "/api/v1/stix/CVE-2026-XXXXX" }
