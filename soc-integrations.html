Microsoft Sentinel Integration
Enterprise Required
Push SENTINEL APEX advisories to your Azure Log Analytics workspace as a custom log type SentinelApexThreatIntel_CL. Advisories appear in Sentinel within 60 seconds of new intel being processed.
1
Get your Enterprise API Key
You need an Enterprise-tier API key to register SIEM webhooks.
Upgrade to Enterprise →  Manage Keys
2
Register the Sentinel Webhook
Call the SENTINEL APEX webhook registration endpoint with your Log Analytics workspace URL and Sentinel format.
curl -X POST https://intel.cyberdudebivash.com/api/webhooks/siem \
  -H "Authorization: Bearer YOUR_ENTERPRISE_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://YOUR_WORKSPACE_ID.ods.opinsights.azure.com/api/logs?api-version=2016-04-01",
    "format": "sentinel",
    "filter_severity": "HIGH",
    "secret": "YOUR_LOG_ANALYTICS_WORKSPACE_KEY"
  }'
ℹ Replace YOUR_WORKSPACE_ID and YOUR_LOG_ANALYTICS_WORKSPACE_KEY from Azure Portal → Log Analytics Workspace → Agents → Workspace ID / Primary Key.
3
Verify Intel Arriving in Sentinel
Open Azure Portal → Microsoft Sentinel → Logs and run this KQL query:
SentinelApexThreatIntel_CL
| where TimeGenerated > ago(1h)
| project TimeGenerated, ThreatTitle_s, Severity_s, RiskScore_d, ActorTag_s, KevPresent_b
| order by RiskScore_d desc
✓ If rows appear, your integration is live. SENTINEL APEX pushes new advisories every 15 minutes via cron.
4
Create Sentinel Analytics Rule
Automatically generate incidents for CRITICAL threats:
// Sentinel Analytics Rule — SENTINEL APEX Critical Threat Alert
SentinelApexThreatIntel_CL
| where Severity_s == "CRITICAL" or RiskScore_d >= 8.0
| where KevPresent_b == true or ExploitAvailable_b == true
| extend AlertName = strcat("CRITICAL THREAT: ", ThreatTitle_s)
| project AlertName, ThreatId_s, ActorTag_s, MitreTactics_s, RiskScore_d, ConfidenceScore_d
| order by RiskScore_d desc
ℹ Rule type: Scheduled · Frequency: 15 minutes · Lookback: 30 minutes

Splunk Enterprise / Cloud Integration
Enterprise Required
SENTINEL APEX delivers events via Splunk HTTP Event Collector (HEC) in native Splunk JSON format. Events are indexed in threat_intel with sourcetype cyberdude:sentinel:advisory.
1
Enable HTTP Event Collector in Splunk
Navigate to Settings → Data Inputs → HTTP Event Collector → Global Settings. Enable HEC and note your HEC token.
⚠ Create a dedicated HEC token for SENTINEL APEX. Set index to threat_intel and sourcetype to cyberdude:sentinel:advisory.
2
Register the Splunk Webhook
curl -X POST https://intel.cyberdudebivash.com/api/webhooks/siem \
  -H "Authorization: Bearer YOUR_ENTERPRISE_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://YOUR_SPLUNK_HOST:8088/services/collector/event",
    "format": "splunk",
    "filter_severity": "HIGH",
    "secret": "Splunk YOUR_HEC_TOKEN"
  }'
3
Verify in Splunk Search
index=threat_intel sourcetype="cyberdude:sentinel:advisory"
| eval risk=round(event.risk_score,1)
| sort -risk
| table _time, event.title, event.severity, risk, event.actor, event.kev, event.ioc_count
✓ Events arriving with sourcetype cyberdude:sentinel:advisory confirm successful integration.
4
Create Splunk Alert
Set up a real-time alert for CRITICAL threats in ESCU (Enterprise Security Content Updates):
index=threat_intel sourcetype="cyberdude:sentinel:advisory"
| where event.severity="CRITICAL" OR event.risk_score>=8
| eval urgency=case(event.kev="true","IMMEDIATE",event.risk_score>=9,"CRITICAL",1=1,"HIGH")
| table _time, event.title, urgency, event.actor, event.ioc_count, event.mitre_tactics
| sort -event.risk_score
ℹ Alert type: Real-time · Trigger: Per result · Action: Create notable event / PagerDuty / Slack

IBM QRadar Integration
Enterprise Required
SENTINEL APEX pushes advisories in LEEF 2.0 syslog format. QRadar parses events automatically using the SENTINEL APEX DSM.
1
Configure QRadar Log Source
In QRadar Console: Admin → Log Sources → Add. Set type to Universal DSM. Note the syslog listener port (default: 514/UDP or 6514/TCP).
⚠ For production use TCP/6514 (TLS). Open firewall from SENTINEL APEX Worker IPs to your QRadar Console.
2
Register QRadar Webhook
curl -X POST https://intel.cyberdudebivash.com/api/webhooks/siem \
  -H "Authorization: Bearer YOUR_ENTERPRISE_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://YOUR_QRADAR_HOST:443/api/reference_data/sets/sentinel_apex_iocs",
    "format": "qradar",
    "filter_severity": "HIGH"
  }'
3
Sample LEEF 2.0 Event Format
Events pushed by SENTINEL APEX look like this in QRadar:
LEEF:2.0|CYBERDUDEBIVASH|SENTINEL-APEX|143.0.0|ThreatIntel|
  devTime=2026-05-03T12:00:00Z	devTimeFormat=ISO 8601	cat=ThreatIntel
  sev=9.2	ThreatTitle=Critical RCE in Apache Struts	Severity=CRITICAL
  ActorTag=APT28::H::IOC-12::TTP-8	IocCount=12	KevPresent=true
  ConfidenceScore=91	FeedSource=NVD-KEV	GatewayVersion=SENTINEL-APEX/143.0.0
✓ QRadar auto-maps LEEF fields to offense rules. Review Offenses tab after first push.

Elastic SIEM / Security Integration
Pro+
Poll the SENTINEL APEX feed API and ingest directly into an Elasticsearch index using Filebeat or a Python ingestor. ECS-compatible JSON structure for native Elastic Security field mapping.
1
Polling Script (Python)
Run this on a scheduled cron every 15 minutes to ingest new advisories:
import requests, json
from elasticsearch import Elasticsearch

APEX_API = "https://intel.cyberdudebivash.com"
APEX_KEY  = "YOUR_PRO_OR_ENT_KEY"
ES_HOST   = "https://your-elastic-host:9200"
ES_INDEX  = "sentinel-apex-threat-intel"
ES_API_KEY= "YOUR_ELASTIC_API_KEY"

es   = Elasticsearch(ES_HOST, api_key=ES_API_KEY, verify_certs=True)
feed = requests.get(f"{APEX_API}/api/feed",
         headers={"Authorization": f"Bearer {APEX_KEY}"},
         params={"limit": 500}).json()

for item in feed.get("advisories", []):
    doc = {
        "@timestamp": item.get("processed_at"),
        "threat.indicator.type":  item.get("type", "unknown"),
        "threat.tactic.name":     item.get("mitre_tactics", []),
        "vulnerability.id":       item.get("cve_id"),
        "vulnerability.score.base": item.get("cvss_score"),
        "event.severity":         item.get("risk_score", 0),
        "event.category":         "threat",
        "sentinel_apex": item,
    }
    es.index(index=ES_INDEX, id=item["id"], body=doc)

print(f"Indexed {len(feed.get('advisories',[]))} advisories")
2
Create Kibana Detection Rule
In Elastic Security → Detections → Create rule → Custom query:
sentinel_apex.severity: "CRITICAL" AND sentinel_apex.kev_present: true
ℹ Set rule schedule to every 15 minutes, severity: Critical, actions: PagerDuty / Slack / webhook.

MISP Threat Sharing Platform
Enterprise Required
Export SENTINEL APEX IOCs and advisories directly into MISP as events and attributes. Uses the native MISP JSON format via GET /api/export/misp.
1
Fetch MISP JSON Export
curl -H "Authorization: Bearer YOUR_ENTERPRISE_KEY" \
  "https://intel.cyberdudebivash.com/api/export/misp?limit=100&since=24h" \
  -o sentinel_apex_misp.json
2
Import into MISP via API
curl -X POST https://YOUR_MISP_HOST/events/add \
  -H "Authorization: YOUR_MISP_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  --data @sentinel_apex_misp.json
✓ MISP events created with SENTINEL APEX source tag and full IOC attribute set.

Generic REST / Webhook Integration
Pro+
Any platform supporting inbound webhooks, REST APIs, or STIX 2.1 feeds can integrate with SENTINEL APEX using the raw JSON or STIX format.
1
Poll the Feed API
# Get latest 100 advisories
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://intel.cyberdudebivash.com/api/feed?limit=100&severity=HIGH,CRITICAL"

# Get only CVEs
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://intel.cyberdudebivash.com/api/cves?kev_only=true&min_epss=0.5"

# Bulk CSV export (Pro+)
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://intel.cyberdudebivash.com/api/export/csv?since=24h" \
  -o threat_intel_$(date +%Y%m%d).csv
2
Register Push Webhook (Enterprise)
curl -X POST https://intel.cyberdudebivash.com/api/webhooks/siem \
  -H "Authorization: Bearer YOUR_ENTERPRISE_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-platform.com/webhook/threat-intel",
    "format": "json",
    "filter_severity": "HIGH",
    "secret": "your-hmac-signing-secret"
  }'
ℹ Payload signature: SENTINEL APEX sends X-APEX-Signature: sha256=HMAC_HEX for webhook verification.
