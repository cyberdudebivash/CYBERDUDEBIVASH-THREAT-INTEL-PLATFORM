Section 02
Security Questionnaire — Pre-filled Responses
Standard CAIQ / SIG Lite / NIST-aligned security questions with responses. Copy directly into your vendor assessment form.
QuestionResponse
ACCESS CONTROL
Does the platform support role-based access control (RBAC)? ✓ YES
Enterprise and MSSP tiers include RBAC with Admin, Analyst, Read-Only, and MSSP-Tenant roles. API key scopes enforce least-privilege access per credential.
Is multi-factor authentication (MFA) supported? ✓ YES
TOTP-based MFA available for all user accounts. Enterprise accounts can enforce MFA organization-wide via admin panel.
Are API keys scoped and revocable? ✓ YES
All API keys carry a scope bitmap (read/write/admin), expiry date, and IP allowlist. Keys revocable instantly from the API Key Manager dashboard.
Is SSO / SAML 2.0 supported for Enterprise accounts? ⚡ ROADMAP
SAML 2.0 SSO integration is on the Enterprise roadmap for Q3 2026. Contact enterprise team for early-access participation.
DATA SECURITY & ENCRYPTION
Is data encrypted in transit? ✓ YES
All traffic uses TLS 1.2+ (TLS 1.3 preferred). HTTP is permanently redirected to HTTPS. HSTS enforced with preload.
Is data encrypted at rest? ✓ YES
AES-256 encryption at rest for all stored intelligence data, API keys (hashed with bcrypt), and customer metadata.
Does the platform store customer-uploaded data? ✓ LIMITED
SENTINEL APEX is primarily a read/API platform. Customer data stored is limited to: account credentials, API key hashes, billing metadata, and optional webhook configurations. No customer threat data is stored unless explicitly submitted for enrichment.
What is the key management practice? API keys are stored as bcrypt hashes (cost factor 12). No plaintext secrets are persisted post-generation. Customer environment secrets (SIEM webhook URLs) are stored encrypted using AES-256-GCM with envelope key rotation quarterly.
VULNERABILITY MANAGEMENT
Is there a vulnerability disclosure / responsible disclosure policy? ✓ YES
Responsible disclosure contact: security@cyberdudebivash.in. CVSS-based triage. Critical vulnerabilities patched within 48 hours per SLA.
Is dependency scanning performed in CI/CD? ✓ YES
GitHub Actions pipeline includes OWASP dependency-check, pip-audit, and Trivy container scan on every commit. Failures block deployment.
Is there a penetration testing schedule? ⚡ ANNUAL
Annual penetration testing planned. Most recent internal security audit: Q1 2026. Enterprise customers may request audit reports under NDA.
INCIDENT RESPONSE
What is the incident notification SLA for Enterprise customers? ✓ 4 HOURS
Security incidents affecting customer data: notification within 4 business hours. P0 outages: notification within 1 hour via status page and email. See Section 04 for full SLA table.
Is there a documented incident response plan? ✓ YES
NIST SP 800-61 aligned IRP covering: identification, containment, eradication, recovery, and post-incident analysis. Available under NDA for Enterprise accounts.
LOGGING & MONITORING
Are API access logs available to customers? ✓ YES
Enterprise API access logs (timestamps, endpoint, response code, source IP) available via the customer dashboard and API. Retained for 90 days.
Is there centralized log aggregation and anomaly detection? ✓ YES
All platform events are aggregated with automated anomaly detection. Alerting on: abnormal API volume, auth failures, geographic anomalies, and dependency failures.
SUPPLY CHAIN & THIRD PARTIES
What third-party services process customer data? Minimal third-party data exposure. Payment processing: third-party payment gateway (customer billing data only, PCI-scoped). Analytics: privacy-respecting first-party analytics only. No customer intelligence data is shared with third parties.
Are third-party threat intelligence sources vetted? ✓ YES
All 74 integrated feeds are evaluated for: source reputation, language/quality filtering, false-positive rate, and TLP compliance. Feeds are reviewed quarterly and low-quality sources removed (e.g., NCSC-NL Dutch-language feed removed v185.0).
