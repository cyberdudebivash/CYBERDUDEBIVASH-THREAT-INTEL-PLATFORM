Quick Start Guide

Step 1: Get Your API Key

Email iambivash.bn@gmail.com with your desired tier. API keys are provisioned within 24 hours. Keys follow the format cdb__<32chars>.

Step 2: Make Your First Request

curl -H "X-API-Key: YOUR_API_KEY" \
  "https://intel.cyberdudebivash.com/api/feed?limit=5&severity=CRITICAL"

Step 3: Download STIX Bundle

curl -H "X-API-Key: YOUR_API_KEY" \
  "https://intel.cyberdudebivash.com/api/stix/bundle" \
  -o sentinel_apex_bundle.json

Step 4: Set Up TAXII Pull (Enterprise/MSSP)

taxii2-client \
  --url https://intel.cyberdudebivash.com/taxii/cdb-apex/ \
  --collection sentinel-apex-live \
  --header "Authorization: Bearer YOUR_API_KEY"

Step 5: Parse Advisory with Python

import requests

headers = {"X-API-Key": "YOUR_API_KEY"}
feed = requests.get("https://intel.cyberdudebivash.com/api/feed", headers=headers).json()

for advisory in feed:
    print(f"[{advisory['severity']}] {advisory['title']}")
    print(f"  CVSS: {advisory.get('cvss_score','N/A')} | EPSS: {advisory.get('epss_score','N/A')}%")
    print(f"  CVEs: {', '.join(advisory.get('cve_ids', []))}")
    print(f"  Report: {advisory.get('report_url','')}")
    print()

SIEM Integration Guides

Microsoft Sentinel (TAXII Data Connector)

  1. In Sentinel → Data Connectors → search "TAXII"
  2. Click "Open connector page" → "Add new"
  3. Friendly name: SENTINEL APEX CTI
  4. API root URL: https://intel.cyberdudebivash.com/taxii/cdb-apex/
  5. Collection ID: sentinel-apex-live
  6. Username: api | Password: YOUR_API_KEY
  7. Import interval: 1 hour

After 1 cycle, threat indicators appear in ThreatIntelligenceIndicator table.

Splunk Enterprise Security

# In inputs.conf:
[sentinel_apex_taxii]
interval = 3600
index = threat_intel
sourcetype = stix
url = https://intel.cyberdudebivash.com/api/stix/bundle
http_method = GET
auth_type = bearer_token
bearer_token = YOUR_API_KEY
polling_interval = 3600

Map to Splunk's threat_intel index. Use | tstats count WHERE index=threat_intel to verify ingestion.

IBM QRadar

Configure the TAXII 2 Feed integration in XSOAR:

MSSP Onboarding Guide

Architecture Overview

MSSP deployments use a single master API key with per-client subkeys. Each subkey is rate-limited independently, enabling per-client usage billing.

Client Provisioning

  1. Submit client list to iambivash.bn@gmail.com with client names and desired rate limits
  2. Receive per-client API keys (format: cdb_mssp_client_{n}_{32chars})
  3. Each client key is independently rate-limited and usage-tracked
  4. Monthly usage reports emailed to MSSP billing contact

White-Labeling

Contact us for white-label API wrapper configuration. SENTINEL APEX supports custom API subdomain routing (e.g., threat-intel.yourcompany.com/api/) via CNAME + custom Cloudflare Worker routing.

Use Case Library

Threat Hunting: Find Unpatched CVEs in Your Environment

import requests

headers = {"X-API-Key": "YOUR_API_KEY"}
feed = requests.get(
    "https://intel.cyberdudebivash.com/api/feed?severity=CRITICAL",
    headers=headers
).json()

# Enrich with your asset inventory
for adv in feed:
    for cve in adv.get("cve_ids", []):
        # Query your CMDB/vulnerability scanner for this CVE
        print(f"Hunt: {cve} | EPSS: {adv.get('epss_score')}% | Actor: {adv.get('threat_actor')}")

Automated Vulnerability Prioritization (EPSS + CVSS + KEV)

priority_score = (
    advisory["cvss_score"] * 0.4 +
    advisory["epss_score"] * 0.4 +
    (20 if advisory.get("kev_present") else 0)
)
# Score > 15: Patch immediately
# Score 10-15: Patch within 72h
# Score < 10: Patch in next cycle

SOAR Playbook Trigger (MITRE ATT&CK)

Use MITRE technique IDs from the feed to automatically trigger hunt playbooks:

PLAYBOOKS = {
    "T1190": "playbook_external_exploit",
    "T1078": "playbook_credential_compromise",
    "T1486": "playbook_ransomware_response",
}
for technique in advisory.get("mitre_techniques", []):
    if technique in PLAYBOOKS:
        soar.trigger(PLAYBOOKS[technique], advisory)
