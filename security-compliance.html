PLATFORM SECURITY

Security Architecture

🔐 API Authentication & Key Management

All API access is protected by HMAC-SHA256 signed API keys with configurable scopes and rate limits. Keys are bcrypt-hashed (cost 12) at rest and never logged in plain text. Enterprise accounts support IP allowlisting and zero-downtime key rotation.

  • HMAC-SHA256 signed API keys
  • bcrypt cost-12 key storage at rest
  • Per-key scope restrictions (read / export / admin)
  • IP allowlisting for Enterprise tier (CIDR ranges)
  • Key rotation with 24h grace period (zero downtime)
  • Rate limiting: 100 / 5,000 / unlimited by tier

🛡️ Transport & Data Security

All API endpoints and dashboard routes are served exclusively over TLS 1.3 (TLS 1.2 deprecated Q4 2025). HSTS headers enforce HTTPS. Content Security Policy prevents XSS. Data at rest is encrypted using AES-256.

  • TLS 1.3 minimum (TLS 1.2 deprecated 2025-Q4)
  • HSTS enforced (max-age=31536000; includeSubDomains)
  • Content Security Policy Level 3
  • AES-256 encryption at rest
  • Subresource Integrity on all CDN assets
  • X-Frame-Options DENY · X-Content-Type-Options nosniff

📡 Feed Data Integrity

Every published feed item carries a SHA-256 content hash and a schema validation timestamp. The IOC Quality Governor v1.1 runs artifact rejection, confidence validation, and ATT&CK precision checks before any item is published.

  • SHA-256 per-item content hash
  • IOC artifact gate (code-file / version string rejection)
  • Confidence inflation prevention
  • ATT&CK tactic/technique ID format validation (v15)
  • False-positive rate monitoring target <5%
  • Actor attribution evidence gates

🔒 Access Control & MFA

Role-based access controls govern what each API key and user session can access. MSSP tenants are fully isolated — no cross-tenant data access is architecturally possible. MFA is enforced on all admin accounts. SSO (SAML 2.0 / OIDC) is on the Q3 2026 roadmap.

  • RBAC with role-scoped API keys
  • MSSP tenant hard isolation (no cross-tenant reads)
  • MFA enforced for admin and MSSP-admin roles
  • Admin elevated session tokens (15-min TTL)
  • Audit log for all privileged actions (90-day retention)
  • SSO (SAML 2.0 / OIDC) — roadmap Q3 2026

TLP CLASSIFICATION

Traffic Light Protocol Compliance

TLP:CLEAR

Unrestricted. Public intelligence. Freely shareable. Used for all non-sensitive vulnerability advisories and publicly-disclosed IOCs. Available on Community tier.

TLP:GREEN

Community-restricted. Shareable within the security community. Used for campaign intelligence with limited attribution confidence. PRO tier and above.

TLP:AMBER

Limited distribution. Recipient organisation only. High-confidence IOC sets and actor attribution with source protection. Enterprise tier and above.

TLP:RED

Named recipients only. Enterprise and MSSP partner exclusive. Not for redistribution. TLP:RED intelligence is never displayed on public dashboards (FIRST TLP spec §2.4).

INCIDENT RESPONSE

Incident Response & Notification SLA

All security incidents are handled under NIST SP 800-61 (Computer Security Incident Handling Guide). Notification timelines below apply to Enterprise and MSSP plan holders.

  • T+0 — Detection
    Automated monitoring detects anomaly or security event
    45-stage CI/CD pipeline with health checks; uptime monitoring alerts on-call engineer within 2 minutes of threshold breach.
  • T+15 min — Initial Triage
    On-call engineer performs initial triage and severity classification
    Severity levels: CRITICAL (platform down / active breach), HIGH (data integrity / access control), MEDIUM (degraded performance), LOW (cosmetic / logging).
  • T+4 hours — Customer Notification (SLA)
    HIGH and CRITICAL security incidents: customer notification
    Enterprise and MSSP customers notified within 4 hours via email and (if configured) webhook. Notification includes: incident summary, affected scope, current status, and preliminary remediation steps.
  • T+15 min — CRITICAL Platform SLA
    CRITICAL advisory delivery to MSSP tenants
    CRITICAL-severity threat intelligence pushed to MSSP partner webhooks within 15 minutes of pipeline publish. Covered under MSSP SLA contract.
  • T+72 hours — RCA Report
    Root cause analysis and post-incident report
    Full RCA delivered to affected Enterprise/MSSP customers: root cause, timeline, remediation steps taken, and preventive controls added.
Uptime SLA by tier: Community — best effort · PRO — 99.5% monthly · Enterprise — 99.9% monthly · MSSP — 99.9% monthly with 15-min CRITICAL SLA. SLA credits apply per contract terms. Email enterprise@cyberdudebivash.com for SLA documentation.
VULNERABILITY DISCLOSURE

Responsible Disclosure Policy

🔍 Report a Security Vulnerability

We welcome responsible disclosure from security researchers. If you discover a vulnerability in SENTINEL APEX, the API, or any infrastructure at intel.cyberdudebivash.com, please report it to us before public disclosure so we can triage and remediate.

Contact: Email security@cyberdudebivash.com with subject line [VULN] Brief description. For high-severity findings, use PGP encryption — public key available at /.well-known/security.txt.

What to include: Affected endpoint or component, reproduction steps, impact assessment, and any proof-of-concept (non-destructive). We do not currently operate a bug bounty program but we do provide public acknowledgment (if desired) and will work collaboratively on remediation.

Our commitments: Acknowledge within 2 business days. Provide triage outcome within 7 business days. Notify you before deploying the fix. Request coordinated disclosure before any public communication.

Safe harbor: Good-faith security research will not result in legal action. Testing must be against your own accounts and must not include DoS, data exfiltration from other tenants, or social engineering.
