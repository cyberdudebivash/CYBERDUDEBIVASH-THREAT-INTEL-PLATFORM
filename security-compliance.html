PLATFORM SECURITY

Security Architecture

🔐 API Authentication & Key Management

All API access is protected by HMAC-signed API keys with configurable scopes and rate limits. Keys are never logged in plain text. Enterprise accounts support IP allowlisting and key rotation with zero-downtime.

  • HMAC-256 signed API keys
  • Per-key scope restrictions (read / export / admin)
  • IP allowlisting for Enterprise tier
  • Key rotation with 24h grace period
  • Rate limiting: 100 / 5,000 / unlimited by tier

🛡️ Transport & Data Security

All API endpoints and dashboard routes are served exclusively over TLS 1.3. HSTS headers enforce HTTPS for all browsers. Content Security Policy prevents XSS. No mixed content is permitted.

  • TLS 1.3 minimum (TLS 1.2 deprecated)
  • HSTS enforced (max-age=31536000)
  • Content Security Policy Level 3
  • Subresource Integrity on all CDN assets
  • X-Frame-Options DENY

📡 Feed Data Integrity

Every published feed item carries a SHA-256 content hash and a schema validation timestamp. The Quality Enforcement Engine runs IOC integrity checks, confidence validation, and ATT&CK precision validation before any item is published.

  • SHA-256 per-item content hash
  • IOC contamination detection (code-file artifact removal)
  • Confidence inflation prevention
  • ATT&CK tactic/technique ID validation
  • Actor attribution evidence gates

🔒 Access Control

Role-based access controls govern what each API key and user session can access. MSSP tenants are fully isolated — no cross-tenant data access is architecturally possible. Admin actions require elevated session tokens.

  • Role-based API scopes
  • MSSP tenant isolation (hard boundary)
  • Admin elevated session tokens
  • Audit log for all privileged actions
  • Session timeout enforcement
TLP CLASSIFICATION

Traffic Light Protocol Compliance

TLP:CLEAR

Unrestricted. Public intelligence. Freely shareable. Used for all non-sensitive vulnerability advisories and publicly-disclosed IOCs.

TLP:GREEN

Community-restricted. Shareable within the security community. Used for campaign intelligence with limited attribution confidence.

TLP:AMBER

Limited distribution. Recipient organisation only. Used for high-confidence IOC sets and actor attribution with source protection.

TLP:RED

Named recipients only. Enterprise and MSSP partner exclusive. Not for redistribution. Used for sensitive actor intelligence.

STANDARDS

Industry Standards Conformance

All threat intelligence is modelled and exportable in STIX 2.1 format. Bundles include Indicators, Malware, Attack Pattern, and Threat Actor objects.
