## What's New

### 1. Dark Web Monitor + Leak Check Engine (`dark-web-monitor.js`)

New backend module powering the Dark Web surveillance feature visible in the dashboard UI.

**Endpoints added:**
- `POST /api/dark-web/scan` — Full breach scan for email address or domain. Checks 7 OSINT/passive intelligence sources including HaveIBeenPwned-compatible, Intelligence X, LeakIX, Shodan Exposure Monitor, Telegram leak channels (Enterprise full scan). Returns `risk_score`, `threat_indicators[]`, `breach_records[]`, `recommended_actions[]`.
- `GET /api/dark-web/status` — Monitor health dashboard. Shows source connectivity, last breach indexed, scan throughput, uptime.
- `GET|POST /api/leak-check` — Quick single email/domain breach check. Returns `pwned`, `breach_count`, `severity`, `recommendation`.

**Tier gates:**
- Free: upgrade CTA, no scan access
- Pro: 50 scans/day, 4 sources, quick scan only
- Enterprise: 500 scans/day, all 7 sources, full deep scan

**Security:** All inputs null-safe (`safeEmail()`, `safeDomain()`). Results KV-cached (15–30 min TTL). No active dark web crawling — passive OSINT aggregation only.

### 3. apex_ai Overwrite — Permanently Fixed (v145.0 Guard)

**Root cause (resolved in prior sprint, now permanently documented):**  
`applyTierGate()` called `computeApexAI(item, tier)` unconditionally. `computeApexAI()` had `toUpperCase()` on potentially-undefined fields.

**Fix (confirmed live in v142.3.1, preserved in v143.0.0):**  
```js
const hasValidApexAI =
  existingApexAI &&
  typeof existingApexAI === "object" &&
  !existingApexAI.error &&
  (existingApexAI.ai_summary || typeof existingApexAI.predictive_risk === "number");

if (hasValidApexAI) {
  // Return existing apex_ai (source of truth). Never recompute.
  return { ...existingApexAI };
}
```

All string operations in `computeApexAI()` are null-safe:  
`(item.severity || "UNKNOWN").toUpperCase()` — zero unhandled TypeErrors.

**Status:** CLOSED. No regression.
