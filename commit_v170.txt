fix(RC7): eliminate Chrome GPU pre-promotion trap via backface-visibility -- v170

ROOT CAUSE (FORENSICALLY CONFIRMED):
compositor-governance-engine.js init() and safeMode() were setting
backface-visibility:hidden WITHOUT !important. This property is a confirmed
Chrome GPU compositor layer promotion trigger -- same class as RC3
(will-change:transform) but via a different CSS property that survived all
previous fix iterations RC1-RC6.

Chrome pre-promotes the canvas to an EMPTY compositor layer BEFORE the first
paint, capturing a blank frame. Edge does NOT promote on backface-visibility
alone when will-change:auto is set. Chrome always does. This explains why the
threat map was blank in Chrome Desktop but visible in Edge.
