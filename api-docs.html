SENTINEL APEX API Reference

The SENTINEL APEX API provides programmatic access to real-time threat intelligence, STIX 2.1 bundles, analyst dossiers, and IOC lookups. The API base URL is:

https://intel.cyberdudebivash.com/api/

📘 All API responses are JSON. STIX endpoints additionally return STIX 2.1-compliant bundles. Authentication is required for all endpoints except /api/health.

Authentication

All authenticated endpoints require an API key passed in the X-API-Key header. Request your API key at pricing.html.

curl -H "X-API-Key: YOUR_API_KEY" \
  https://intel.cyberdudebivash.com/api/feed

⚠️ Keep your API key secret. Do not expose it in client-side code. Rotate compromised keys immediately by emailing iambivash.bn@gmail.com.

GET /api/feed

Returns the latest threat intelligence feed as a JSON array of advisories.

GET/api/feed

Query Parameters

ParameterTypeRequiredDescription
limitintegeroptionalMax results (default: 20, max: 100)
severitystringoptionalFilter: CRITICAL, HIGH, MEDIUM, LOW
sinceISO-8601optionalReturn advisories after this timestamp
actorstringoptionalFilter by threat actor name

GET /api/stix/bundle

Returns a STIX 2.1 compliant bundle of all current threat intelligence. Compatible with all STIX-aware SIEMs.

GET/api/stix/bundle
curl -H "X-API-Key: YOUR_API_KEY" \
  https://intel.cyberdudebivash.com/api/stix/bundle \
  -o stix_bundle.json

Splunk Integration

Install the SENTINEL APEX Technology Add-on for Splunk from Splunkbase (submission in progress) or configure manually:

[sentinel_apex_feed]
interval = 3600
sourcetype = stix
url = https://intel.cyberdudebivash.com/api/stix/bundle
http_method = GET
auth_type = bearer
token = YOUR_API_KEY

Microsoft Sentinel Integration

Use the TAXII Data Connector in Microsoft Sentinel:

  1. In Sentinel → Data Connectors → search "TAXII"
  2. Add connector: Server URL = https://intel.cyberdudebivash.com/taxii/
  3. API Root: https://intel.cyberdudebivash.com/taxii/cdb-apex/
  4. Collection ID: sentinel-apex-live
  5. Authentication: HTTP Header — Authorization: Bearer YOUR_API_KEY

IBM QRadar Integration

Configure a STIX-TAXII reference set source in QRadar:

qradar-taxii-client \
  --server https://intel.cyberdudebivash.com/taxii/cdb-apex/ \
  --collection sentinel-apex-live \
  --api-key YOUR_API_KEY \
  --output-type stix2
