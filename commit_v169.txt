RC6 ROOT CAUSE (Chrome blank map — forensically confirmed):
  Line 14878: v159 CSS sets will-change:transform on #cdb-threat-canvas (no !important)
  Chrome GPU compositor reads this rule at CSS parse time and PRE-PROMOTES canvas
  to a separate GPU compositor layer BEFORE any JS runs. When JS later sets
  will-change:auto (RC3), Chrome has a compositor conflict: promoted layer + auto
  hint = blank canvas. Edge compositor handles this conflict differently = renders.

RC6 FIX (index.html):
  Immediately after v159 rule: #cdb-threat-canvas { will-change:auto !important;
  transform:none !important } — Chrome sees auto as the final cascaded value.
  No pre-promotion occurs. JS post-paint promotion (RC3+G4) proceeds cleanly.
