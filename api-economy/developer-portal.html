Sentinel APEX API Documentation

The Sentinel APEX API is a RESTful threat intelligence platform API with STIX 2.1 native output, real-time IOC lookups, advisory feeds, risk scoring, ATT&CK mapping, and proprietary telemetry ingest. All responses are JSON. Base URL: https://intel.cyberdudebivash.com/api/v3

Quick Start
bash
# Get your first advisory in 30 seconds
curl -X GET \
  "https://intel.cyberdudebivash.com/api/v3/advisories?severity=high&limit=5" \
  -H "Authorization: Bearer sk-cdb-YOUR_API_KEY" \
  -H "Accept: application/json"

# Response: STIX 2.1 advisory bundle with risk scores, IOCs, ATT&CK TTPs
Core Concepts
Every advisory returned by the API is enriched with our proprietary scoring pipeline: evidence-gated density calibration (Tier 0–3), telemetry-weighted confidence with provenance lineage, KEV promotion, and MITRE ATT&CK v16 technique mapping. Our anti-synthetic confidence rule ensures no advisory claims >25% confidence without sufficient telemetry evidence.
Authentication
Python
import sentinel_apex

API Versioning
The current stable version is v3. Previous versions v1 and v2 are deprecated and will be sunset 2026-09-01. Include the version in all URLs: /api/v3/
Authentication

All API requests require a Bearer token in the Authorization header. API keys are scoped per tenant, never cross-tenant accessible, and rotate every 90 days by default.

API Key Types
READ Intelligence Read Key Prefix: sk-cdb-r- | Grants advisory, IOC, risk score reads
INGEST Telemetry Ingest Key Prefix: sk-cdb-i- | Grants telemetry submission endpoints only
ADMIN Admin Key Prefix: sk-cdb-a- | Full platform control. MSSP-only.
Key Rotation
bash
# Rotate API key (generates new key, old key active for 24h grace period)

  -H "Authorization: Bearer sk-cdb-YOUR_KEY" \
  -d '{"grace_period_hours": 24}'

Python SDK — Full Example
Python
from sentinel_apex import Client, AdvisoryFilter, IOCBatch
import asyncio

client = Client(
    api_key="sk-cdb-YOUR_KEY",
    base_url="https://intel.cyberdudebivash.com/api/v3",
    timeout=30,
    retries=3,
    retry_backoff=1.5
)

# Stream advisories in real-time
async def stream_intel():
    async for advisory in client.advisories.stream(
        filter=AdvisoryFilter(severity=["high", "critical"], kev_only=True)
    ):
        print(f"{advisory.cve_id}: {advisory.risk_score:.1f}/10 [{advisory.confidence_pct:.0f}% conf]")
        if advisory.iocs:
            ioc_results = await client.ioc.bulk_lookup_async(advisory.iocs)
            for ioc in ioc_results:
                print(f"  IOC: {ioc.value} → {ioc.verdict} ({ioc.confidence_pct:.0f}%)")

asyncio.run(stream_intel())

Webhook Signature Verification
Python
import hmac, hashlib

def verify_webhook(payload_bytes, signature_header, secret):
    expected = hmac.new(secret.encode(), payload_bytes, hashlib.sha256).hexdigest()
    return hmac.compare_digest(f"sha256={expected}", signature_header)

# In your webhook handler (Flask example):
from flask import request, abort
@app.route('/webhook', methods=['POST'])
def handle_webhook():
    sig = request.headers.get('X-CDB-Signature')
    if not verify_webhook(request.data, sig, WEBHOOK_SECRET):
        abort(401)
    event = request.get_json()
    process_event(event)
    return '', 200

STIX 2.1 Export

All advisory and IOC data is available in STIX 2.1 format. Use the Accept: application/stix+json header or the format=stix query parameter.

bash
curl https://intel.cyberdudebivash.com/api/v3/advisories/CVE-2026-6565 \
  -H "Authorization: Bearer sk-cdb-YOUR_KEY" \
  -H "Accept: application/stix+json"

# Returns STIX Bundle with:
# - vulnerability object (CVE ID, CVSS, EPSS)
# - indicator objects (IOCs with pattern expressions)
# - attack-pattern objects (ATT&CK techniques)
# - relationship objects (linking indicators → attack-patterns)
Rate Limits

Rate limits are enforced per API key using a sliding window algorithm. Headers returned on every response:

HTTP Headers
X-RateLimit-Limit: 5000
X-RateLimit-Remaining: 4847
X-RateLimit-Reset: 1716825600
X-RateLimit-Window: 86400
Retry-After: 0  (only on 429)
Error Handling
200 Success
400 Bad Request — Invalid parameters. Check error.details
401 Unauthorized — Invalid or expired API key
429 Rate Limited — Respect Retry-After header
500 Internal Error — Logged. Contact support@cyberdudebivash.com
